Windows Setup Guide — Dual MCP

Google Search Console
Dual MCP for Claude Desktop

Two MCP servers running side by side — deep per-site analytics AND multi-account management, both accessible in a single Claude Desktop chat.

gsc-deep — Deep analytics via uvx + OAuth credentials
gsc-multi — Multi-account via node, built-in OAuth
Before you start — three placeholders you must replace in the snippets below:
  • YOUR_USERNAME — your Windows username. Open Command Prompt and run echo %USERNAME% to find it. The username goes into every path that starts with C:\Users\.
  • your-google-email@gmail.com — the Gmail you want to authenticate with Google Search Console. Use the same address you use when signing into search.google.com/search-console.
  • YOUR_ALIAS — (gsc-multi only) the short nickname you give each Google account when running gsc-mcp-auth --alias <name>. Lowercase letters, digits, hyphens, underscores only — no spaces, no @. Example aliases: account1, client-acme, personal.

Important: the JSON config file does not expand environment variables — %USERPROFILE% and ~ are interpreted literally. You must write the full path with your real username.

Apply the default patch — show access levels in list_sites

By default, list_sites only prints URLs. This one-time patch (3 string replacements across 2 files inside the installed package) annotates each site with its access level: [siteOwner] / [siteFullUser] / [siteRestrictedUser] / [siteUnverifiedUser]. Run it now — the rest of this guide assumes it's applied.

PowerShell — paste the whole block and press Enter
How to open: Press Win + R → type powershell → press Enter
powershell — multi-line paste
$d="$env:APPDATA\npm\node_modules\google-searchconsole-mcp\dist"
$a=Get-Content "$d\tools\search-analytics.js" -Raw
$findA='        \.map\(\(site\) => site\.siteUrl\)\r?\n        \.filter\(\(url\) => url !== undefined\);'
$replA="        .filter((site) => site.siteUrl !== undefined)`r`n        .map((site) => ({`r`n            siteUrl: site.siteUrl,`r`n            permissionLevel: site.permissionLevel || ""Unknown"",`r`n        }));"
$a=[regex]::Replace($a,$findA,$replA)
Set-Content "$d\tools\search-analytics.js" -Value $a -NoNewline
$b=Get-Content "$d\index.js" -Raw
$b=$b.Replace('sites.forEach((s) => lines.push(`  - ${s}`));','sites.forEach((s) => lines.push(`  - ${s.siteUrl}  [${s.permissionLevel}]`));')
$b=$b.Replace('sites.map((s) => `- ${s}`)','sites.map((s) => `- ${s.siteUrl}  [${s.permissionLevel}]`)')
Set-Content "$d\index.js" -Value $b -NoNewline
"Patch applied. The change takes effect after Claude Desktop is fully restarted."
Expected output: the line "Patch applied. The change takes effect after Claude Desktop is fully restarted."
Safe to re-run. The script is idempotent — running it twice does nothing the second time. If you ever run npm update -g google-searchconsole-mcp the patch is overwritten, so paste this same block again afterward.
If PowerShell refuses to run anything due to execution policy, this paste-and-run pattern is still allowed (it's an interactive command, not a script file). The blocked variant is double-clicking a .ps1 file. If you still hit issues, run Set-ExecutionPolicy -Scope CurrentUser RemoteSigned once and retry.
Step 03
Authenticate Your Google Account(s)

⚠ Browser hygiene — do this first or you'll authenticate the wrong account
Chrome or Edge — Incognito / InPrivate window
Gotcha: Google silently uses whichever Gmail you're already signed into in your browser. If you run gsc-mcp-auth --alias personal while signed in as your work Gmail, the helper saves the work account under the personal alias. The bug is invisible — the auth succeeds — you only notice when the data is wrong.

Pick one before each gsc-mcp-auth command:

  1. Easiest: open an Incognito (Chrome) or InPrivate (Edge) window. Ctrl + Shift + N in Chrome, Edge and Ctrl + Shift + P in Mozilla.
  2. Alternative: go to accounts.google.com/Logout first to sign out of all Google accounts in your normal browser.
After the helper prints "Account email: ..." in Command Prompt, verify it matches the Gmail you intended. If it shows the wrong email, delete the token file at C:\Users\YOUR_USERNAME\.gsc-mcp\tokens\<alias>.json and re-run with a clean browser.

gsc-multi cheat sheet
Two rules to keep in mind every time you use gsc-multi.
★ List all your authenticated accounts at any time

Type this in Claude Desktop chat to see every alias and the Gmail it's bound to:

claude prompt
list all my gsc accounts with alias using gsc-multi

Claude will call the list_accounts tool and print a two-column list: alias on the left, email on the right.

★ Always pass the alias, never the email
When running any gsc-multi tool, use the alias exactly as shown in the left column as the account parameter. For example, to query searchconsoleanalyticsaccess4@gmail.com, pass account: "searchconsoleanalyticsaccess4".

If you pass the full email, the server replies "alias not found" and lists the available aliases. The alias is the part before @gmail.com (the convention this guide uses), but it can be any nickname you chose when you ran gsc-mcp-auth --alias <name>.
